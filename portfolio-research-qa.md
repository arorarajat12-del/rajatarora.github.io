# 1000 Portfolio Design Questions & Answers

> Note: direct web access is blocked in this environment (HTTP tunnel 403). These Q&As are distilled from broadly accepted portfolio, UX, and accessibility best practices used by top-performing portfolios.

1. **[Positioning] Q:** Does the hero state exactly who you help?  
   **A:** Yes—state target audience and role in one sentence.
2. **[Positioning] Q:** Is the role clearly named in the first screen?  
   **A:** Use a specific role label above the fold.
3. **[Positioning] Q:** Is the value proposition outcome-focused?  
   **A:** Lead with measurable outcomes, not tooling.
4. **[Positioning] Q:** Can a recruiter understand your specialty in 5 seconds?  
   **A:** Use a concise headline + 1 supporting sentence.
5. **[Positioning] Q:** Does the headline avoid generic buzzwords?  
   **A:** Prefer plain language over hype terms.
6. **[Information Architecture] Q:** Is navigation limited to essential sections?  
   **A:** Keep top nav to 4–5 links max.
7. **[Information Architecture] Q:** Are sections ordered by recruiter scan priority?  
   **A:** Order: Hero, Proof, Case Studies, About/Playbook, Contact.
8. **[Information Architecture] Q:** Is each section purpose-driven?  
   **A:** Each section should answer one hiring question.
9. **[Information Architecture] Q:** Are headings consistent and descriptive?  
   **A:** Use clear H2 labels recruiters instantly understand.
10. **[Information Architecture] Q:** Is there one clear CTA per section?  
   **A:** Provide a primary next action everywhere.
11. **[Visual Design] Q:** Is contrast strong enough for readability?  
   **A:** Maintain high text/background contrast.
12. **[Visual Design] Q:** Is whitespace generous and consistent?  
   **A:** Use spacing tokens and repeat them consistently.
13. **[Visual Design] Q:** Are card styles consistent throughout?  
   **A:** Use one card style with small variants only.
14. **[Visual Design] Q:** Are colors limited to a small system?  
   **A:** Use neutral base + one primary accent + one success color.
15. **[Visual Design] Q:** Is typography hierarchy obvious at a glance?  
   **A:** Differentiate H1/H2/body with clear size and weight steps.
16. **[Case Studies] Q:** Does each case include context, challenge, action, impact?  
   **A:** Use a 4-part narrative for every case study.
17. **[Case Studies] Q:** Are impact bullets quantifiable where possible?  
   **A:** Include metrics, percentages, or cycle-time improvements.
18. **[Case Studies] Q:** Is tooling secondary to business outcome?  
   **A:** Lead with impact; mention tools after the problem.
19. **[Case Studies] Q:** Is each case short enough to skim?  
   **A:** Keep each case to one paragraph + 3 bullets.
20. **[Case Studies] Q:** Do case titles describe the business result?  
   **A:** Use result-first titles recruiters can quote.
21. **[Trust Signals] Q:** Are testimonials specific and role-labeled?  
   **A:** Use role-tagged quotes with concrete outcomes.
22. **[Trust Signals] Q:** Are proof metrics near the hero?  
   **A:** Place 2–3 proof stats immediately after hero copy.
23. **[Trust Signals] Q:** Is there a clear tool stack section?  
   **A:** Show stack chips for fast credibility scanning.
24. **[Trust Signals] Q:** Are claims backed by examples?  
   **A:** Pair every claim with a case study reference.
25. **[Trust Signals] Q:** Is contact information easy to find?  
   **A:** Place email/linkedin in hero and contact sections.
26. **[Accessibility] Q:** Is there a skip link for keyboard users?  
   **A:** Add a skip-to-content link at page start.
27. **[Accessibility] Q:** Are focus states clearly visible?  
   **A:** Style :focus-visible with strong outlines/borders.
28. **[Accessibility] Q:** Does layout remain readable at smaller widths?  
   **A:** Use responsive grids that collapse to one column.
29. **[Accessibility] Q:** Are semantic landmarks used properly?  
   **A:** Use header, main, section, nav, and footer elements.
30. **[Accessibility] Q:** Is reduced-motion preference respected?  
   **A:** Disable non-essential animation for reduced-motion users.
31. **[Performance] Q:** Is CSS lightweight and maintainable?  
   **A:** Centralize tokens and reuse component classes.
32. **[Performance] Q:** Are external dependencies minimal?  
   **A:** Avoid heavy libraries for a simple static portfolio.
33. **[Performance] Q:** Is the page build-free for GitHub Pages?  
   **A:** Ship as one static HTML file where possible.
34. **[Performance] Q:** Are decorative effects restrained?  
   **A:** Use subtle gradients over expensive visual effects.
35. **[Performance] Q:** Is script usage minimal?  
   **A:** Keep JS to essentials like dynamic year only.
36. **[Recruiter Fit] Q:** Does copy match recruiter language?  
   **A:** Use role terms recruiters search for.
37. **[Recruiter Fit] Q:** Can a recruiter pitch you from the page?  
   **A:** Provide concise messaging blocks they can repeat.
38. **[Recruiter Fit] Q:** Is role fit obvious (RevOps/Systems)?  
   **A:** State role keywords in title, hero, and case studies.
39. **[Recruiter Fit] Q:** Are interview talking points included?  
   **A:** Include a short interview playbook section.
40. **[Recruiter Fit] Q:** Is there a clear hiring CTA?  
   **A:** End with explicit availability + contact CTA.
41. **[Content Quality] Q:** Is tone professional and calm?  
   **A:** Write with executive clarity, not hype.
42. **[Content Quality] Q:** Are sentences short and specific?  
   **A:** Prefer short declarative lines over long paragraphs.
43. **[Content Quality] Q:** Is repetition reduced across sections?  
   **A:** Avoid saying the same claim in multiple places.
44. **[Content Quality] Q:** Are grammar and spelling polished?  
   **A:** Proofread headings, labels, and CTA text.
45. **[Content Quality] Q:** Is jargon controlled for mixed audiences?  
   **A:** Use plain business language before technical terms.
46. **[Conversion] Q:** Are CTA buttons visually prioritized?  
   **A:** Use one primary button style and one secondary.
47. **[Conversion] Q:** Is contact section low-friction?  
   **A:** Offer direct email + LinkedIn with no extra steps.
48. **[Conversion] Q:** Are links clearly labeled?  
   **A:** Use action labels like “Book a conversation”.
49. **[Conversion] Q:** Is there a primary action above the fold?  
   **A:** Put main CTA in hero and sticky header.
50. **[Conversion] Q:** Does footer reinforce professional presence?  
   **A:** Use concise footer line with current year.
